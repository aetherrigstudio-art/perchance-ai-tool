## Perchance API ‚Äî CORRECTED UNDERSTANDING

**AUTOMATION.md is wrong.** Deep web research (commit `14e24ba`) found:

> **`/api/downloadGenerator` has a backwards-compatibility guarantee and is
> callable without browser automation.** Three community tools prove the API
> layer (text gen, image gen, local proxy) is accessible programmatically.

This endpoint lets you call a Perchance generator's `generate()` function via
HTTP without a browser. The 403s we hit were on the Perchance.org *page* URLs
(the HTML editor) ‚Äî not on the API endpoint.

**Implications:**
- The current loader approach (paste once, fetch char-wiz-html from GitHub) is
  still the right deploy path for the HTML panel ‚Äî nothing changes there.
- But we could build **automated generation testing** (call the live generator
  via `/api/downloadGenerator` in CI) and **prompt regression checks** without
  any browser automation.
- AUTOMATION.md should be updated to reflect this distinction: editing the HTML
  panel still requires the loader, but the generate() function is callable via
  HTTP.
